# 开发日志 - 阶段二：动静分离（Nginx 托管静态页 + /api 反代）与 Maven Wrapper 补齐

## 1. 阶段目标与完成情况
本阶段聚焦在“动静分离”，目标是：**静态资源由 Nginx 直接提供**，而业务接口请求通过 **`/api/*` 反向代理**转发给后端（默认 `8080`）。

### 已完成的任务
- [x] **Nginx 静态托管**：浏览器访问 `http://localhost` 可加载 `index.html / style.css / app.js`。
- [x] **/api 反代链路**：前端 JS 通过 `POST /api/user/register`、`POST /api/user/login` 调用后端接口（Nginx 将 `/api` 前缀去掉后再转发）。
- [x] **新增 Nginx 回归用例**：提供 `test/test-nginx.http`，便于用 “走 Nginx 的 URL” 做回归测试。
- [x] **补齐 Maven Wrapper**：补全 `.mvn/wrapper/maven-wrapper.properties`，可直接用 `.\mvnw.cmd ...` 运行 Maven（无需本机预装 Maven）。

## 2. 如何验证 Nginx 是否在发挥作用（你自己可复现）
### 2.1 确认 Nginx 容器在跑
在项目根目录执行：

```powershell
cd d:\Repositories\FlashSale-Pro
docker compose ps
```

看到 `flashsale_nginx` 处于 `Up`，且端口有 `80 -> 容器端口` 映射，即表示 Nginx 正在接管 `http://localhost`。

### 2.2 看响应头验证“确实由 Nginx 返回”

```powershell
curl -I http://localhost/
curl -I http://localhost/style.css
```

如果响应头里出现 `Server: nginx/...`，则证明静态资源确实由 Nginx 提供，而不是后端应用直接提供。

## 3. 如何测试 /api 反代（回归用例）
### 3.1 先启动后端（否则 /api 会 502）
- 方式一：IDE 运行主类 `FlashsaleProApplication`（默认 `8080`）
- 方式二：命令行（使用 Maven Wrapper）：

```powershell
cd d:\Repositories\FlashSale-Pro
.\mvnw.cmd spring-boot:run
```

### 3.2 执行回归请求（走 Nginx）
使用 `test/test-nginx.http` 按顺序执行：
- `POST http://localhost/api/user/register`
- `POST http://localhost/api/user/login`

期望现象：
- 能得到后端返回的 JSON（成功或失败都可接受）
- **不应出现 502**（502 一般表示后端没启动/没在 8080）

## 4. 当前项目框架与关键文件职责（写清楚“每个文件做什么”）
### 4.1 后端（Spring Boot + MyBatis）
- `src/main/java/com/flashsale/flashsale_pro/FlashsaleProApplication.java`
  - **作用**：Spring Boot 入口，启动 Web 服务（默认监听 `8080`）。
- `src/main/resources/application.properties`
  - **作用**：后端配置中心，包含 `server.port`、数据库连接、MyBatis 配置等。
- `src/main/java/com/flashsale/flashsale_pro/controller/`
  - **作用**：Controller 层，负责接收 HTTP 请求并调用 Service。
- `src/main/java/com/flashsale/flashsale_pro/service/`
  - **作用**：Service 层，承载业务逻辑（注册冲突校验、登录密码比对、token 返回等）。
- `src/main/java/com/flashsale/flashsale_pro/mapper/`
  - **作用**：Mapper 层（MyBatis），与数据库交互（查询/插入）。
- `src/main/java/com/flashsale/flashsale_pro/entity/`
  - **作用**：实体类（Entity），映射数据库表结构到 Java 对象。
- `src/main/java/com/flashsale/flashsale_pro/common/`
  - **作用**：通用结构与工具（例如统一返回结构 `Result`）。

### 4.2 容器与基础设施（Docker Compose / Nginx）
- `docker-compose.yml`
  - **作用**：一次性编排 PostgreSQL、Redis、Nginx（静态托管 + /api 反代）等服务的启动、端口、挂载。
- `db/schema.sql`（由 compose 挂载进 PostgreSQL 初始化目录）
  - **作用**：数据库初始化建表脚本，容器首次启动时自动执行。
- `nginx/default.conf`
  - **作用**：Nginx 核心配置：
    - `location /`：静态资源分发（从容器内 `/usr/share/nginx/html`）
    - `location /api/`：反向代理到后端 `8080`（并重写路径去掉 `/api` 前缀）

### 4.3 前端静态页面（演示动静分离）
- `frontend-static/index.html`
  - **作用**：演示页面骨架与输入框/按钮。
- `frontend-static/style.css`
  - **作用**：页面基础样式（不依赖构建工具）。
- `frontend-static/app.js`
  - **作用**：用 `fetch('/api/...')` 发起注册/登录请求，并把响应展示到页面上。

### 4.4 测试用例（可回归）
- `test/test.http`
  - **作用**：直连后端 `http://localhost:8080/...` 的注册/登录回归用例。
- `test/test-nginx.http`
  - **作用**：走 Nginx 的 `http://localhost/api/...` 回归用例（用于验证动静分离 + 反代链路）。
