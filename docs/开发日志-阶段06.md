# 开发日志 - 阶段06：智能网关动态拦截与降级策略

## 1. 阶段目标与完成情况

本阶段聚焦在 **FR3（基于意愿得分的智能网关动态拦截）** 与 **NFR1/NFR2（高并发低延迟 / 降级熔断高可用）** 的完整落地。

### 已完成的核心工作

- [x] **IntentFilter 全局过滤器**：`gateway/.../filter/IntentFilter.java`
  - **决策延迟 <5ms**：通过 Reactive Redis 的 `opsForValue().get()` + `.timeout(Duration.ofMillis(5))` 实现
  - **流量染色与分级路由**：
    - HIGH（≥0.7）：优先放行，透传 `X-Intent-Tier: HIGH` 头
    - MEDIUM（0.3~0.7）：正常放行，透传 `X-Intent-Tier: MEDIUM` 头
    - LOW（<0.3）：直接拦截，返回 429 + "购买意愿评分过低"
  - **平滑降级（NFR2）**：
    - Redis 超时/异常 → 自动切换令牌桶限流（`checkTokenBucket` 方法）
    - 无意愿数据 → 降级到令牌桶，保证主干秒杀链路可用
    - 令牌桶参数：capacity=1000, refillRate=100/s
  - **拦截统计**：totalRequests/highIntent/mediumIntent/lowBlocked/degraded 五维计数器
- [x] **Redis Reactive 配置**：`gateway/.../config/RedisConfig.java` + pom.xml 新增 `spring-boot-starter-data-redis-reactive`
- [x] **拦截统计端点**：`gateway/.../controller/IntentStatsController.java` → `GET /actuator/intent/stats`
- [x] **网关配置扩展**：application.yml 新增 intent.* 配置组 + Redis 连接配置
- [x] **docker-compose 更新**：gateway 服务添加 Redis 依赖 + 环境变量
- [x] **白名单更新**：`/actuator/intent` 加入免鉴权白名单

## 4. 给后来者的指导

- IntentFilter 的 order=-1，在 AuthGlobalFilter(order=0) 之前执行，确保已鉴权请求才做意图判断
- Redis 5ms 超时是 NFR1 的硬约束，不可调大，否则会拖垮网关吞吐
- 降级到令牌桶时，capacity 和 refillRate 需根据压测结果调优
- `X-Intent-Tier` 头会透传到下游微服务，order-service 可据此做差异化处理（如高潜用户延长支付超时）
