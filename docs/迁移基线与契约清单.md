# 迁移基线与契约清单（M0）

## 1. 当前能力基线

- 统一入口：`Nginx -> Gateway -> goods/order`
- 秒杀链路：`Redis Lua` 预扣 + `RabbitMQ` 异步下单 + 结果轮询
- 鉴权：`JWT` + 网关透传用户上下文
- 可观测：`X-Trace-Id` + `MDC` + `actuator/prometheus`

## 2. 迁移映射（单体到子模块）

- `user/goods` 相关控制器、服务、Mapper、实体：从 `src/main/java/com/flashsale/flashsale_pro` 迁往 `flashsale-goods-app`
- `order` 相关控制器、服务、MQ、Mapper、实体：迁往 `flashsale-order-app`
- `stock` 相关能力：在 `flashsale-stock` 新增完整闭环（Controller/Service/Mapper/DTO）
- 公共能力（`Result`/限流等）：收敛到 `flashsale-common`

## 3. 契约冻结项

### API 契约

- `POST /api/flash-sale/order`
- `GET /api/flash-sale/order/result`
- `GET /api/flash-sale/items/{itemId}`
- `GET /api/user/me`

### Redis Key 契约

- `seckill:stock:{itemId}`
- `seckill:bought:{itemId}`
- `seckill:result:{userId}:{itemId}`

### MQ 契约

- Exchange: `flashsale.seckill.exchange`
- Queue: `flashsale.seckill.queue`
- RoutingKey: `flashsale.seckill.order`

### 数据库契约

- 订单唯一约束：用户+秒杀商品维度唯一
- 库存扣减必须是条件更新（防超卖）

## 4. 回归检查表

- 模块可独立启动：goods/order/stock/gateway
- 网关路由可用：`/api/**`
- 秒杀并发下：无超卖、无重复单、可查询结果
- 文档、压测脚本、配置与代码一致
