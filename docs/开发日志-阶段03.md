## 2. 当前项目架构概览（再次梳理）

站在“完整秒杀 Demo”的视角，项目可以分为以下几个层次：

### 2.1 前端展示层（静态页面，由 Nginx 托管）
- **技术栈**：原生 HTML + CSS + JS，无打包构建。
- **位置**：`src/main/resources/static/`。
- **主要职责**：
  - 提供用户注册、登录的表单与按钮；
  - 调用 `/api/user/register`、`/api/user/login` 完成基础认证演示；
  - 调用 `/api/flash-sale/events` 查询可用的秒杀活动，展示在页面上；
  - 通过 Nginx 静态托管，实现与后端的 **动静分离**。

### 2.2 网关与反向代理层（Nginx）
- **核心文件**：`nginx/default.conf`。
- **主要职责**：
  - 直接返回静态资源：`index.html`、`style.css`、`app.js`；
  - 将 `/api/*` 请求反向代理到宿主机 `8080` 端口上的 Spring Boot 应用，并去掉 `/api` 前缀；
  - 为后续引入负载均衡（upstream）与缓存策略预留空间。

### 2.3 应用服务层（Spring Boot 后端）
- **入口与配置**：
  - `FlashsaleProApplication`：Spring Boot 启动类；
  - `application.properties`：端口、数据源、MyBatis 配置等；
  - `pom.xml`：依赖管理（Spring Boot、MyBatis、PostgreSQL、测试依赖等），支持通过 `mvnw` 或全局 Maven 运行。
- **通用组件**：
  - `common/Result`：统一返回结果结构；
  - `common/RateLimiterService`：应用层限流组件（内存计数实现），为高频接口提供基础保护。
- **领域与业务模块**：
  - `entity/*`：`User`、`Product`、`FlashSaleEvent`、`FlashSaleItem`、`FlashSaleOrder` 等领域实体；
  - `mapper/*`：各实体对应的 MyBatis Mapper，负责增删改查；
  - `service/*` 与 `service/impl/*`：封装用户、商品、活动、活动商品、订单等业务逻辑（含库存校验、活动状态校验、单用户限购约束等）；
  - `controller/*`：
    - 用户接口：`UserController`；
    - 管理端接口：`AdminProductController`、`AdminFlashSaleEventController`、`AdminFlashSaleItemController`；
    - 用户侧接口：`FlashSaleQueryController`（查询）、`FlashSaleOrderController`（下单 / 查询 / 模拟支付）。

### 2.4 数据存储层（PostgreSQL + 预留 Redis）
- **PostgreSQL**：
  - 通过 `docker-compose.yml` 启动，端口映射为 `15432:5432`；
  - 首次启动时自动执行 `db/schema.sql`，创建 `user`、`product`、`flash_sale_event`、`flash_sale_item`、`flash_sale_order` 五张核心表；
  - 依赖表级约束（外键、唯一约束等）与乐观锁版本号，为业务校验和高并发下的数据一致性打基础。
- **Redis（预留）**：
  - 已在 `docker-compose.yml` 中定义，但当前业务逻辑暂未使用；
  - 未来可以接入：
    - 登录 Token / 会话管理；
    - 活动库存缓存与预减库存；
    - 分布式限流等。

## 3. 端到端调用链示例

以“用户打开页面，查看秒杀活动列表”为例，当前链路如下：

1. 浏览器访问 `http://localhost`；
2. Nginx 返回静态页面 `index.html`，并加载 `style.css` 与 `app.js`；
3. 页面加载完成后，用户点击“加载秒杀活动”按钮，触发 `app.js` 中的 `loadFlashSaleEvents()`；
4. 前端发起 `GET /api/flash-sale/events` 请求；
5. Nginx 将请求反向代理为 `GET /flash-sale/events` 到后端 `8080`；
6. 后端 `FlashSaleQueryController` 接收请求，调用 `FlashSaleEventService` / `FlashSaleItemService` 查询活动与活动商品；
7. Service 层通过 MyBatis 的 `FlashSaleEventMapper`、`FlashSaleItemMapper` 访问 PostgreSQL；
8. 数据查询结果封装为 `Result` 返回给前端，前端将 JSON 渲染在页面中的输出区域。

类似地，在“创建活动 / 配置活动商品 / 用户下单”的链路中，会依次经过：

> 管理端接口（Admin Controller） → Service → Mapper → PostgreSQL  
> 用户端接口（Query & Order Controller） →（可选限流）→ Service → Mapper → PostgreSQL。
