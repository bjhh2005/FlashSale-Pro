# 开发日志 - 阶段05：购买意愿预测模型与SHAP可解释性分析

## 1. 阶段目标与完成情况

本阶段聚焦在 **FR2（离线购买意愿预测与可解释性分析）** 与 **NFR3（模型准确率与可解释性指标）** 的完整落地。

### 已完成的核心工作

- [x] **LightGBM/XGBoost 预测模型**：`bi/app/model/predictor.py`
  - `IntentPredictor` 类：支持 train/predict/predict_single，自动 train/val 分割
  - LightGBM 配置：200 棵树 / 深度 6 / 学习率 0.05 / subsample 0.8
  - XGBoost 配置：同等超参
  - 训练输出 AUC 指标与 classification_report
  - 模型持久化（joblib）+ 特征名 JSON 持久化
- [x] **SHAP 可解释性框架**：`bi/app/model/explainer.py`
  - `ShapExplainer` 类：基于 TreeExplainer（适配 LightGBM/XGBoost）
  - `explain_global`：全局特征重要性排序（mean |SHAP value|）
  - `explain_local`：单用户局部归因分析（正/负贡献 + 基准值 + 预测值）
  - 满足 NFR3"预测模型不能是完全黑盒"的要求
- [x] **FastAPI BI 服务**：`bi/app/api/main.py`
  - `POST /predict`：单用户预测 + Redis 同步
  - `POST /predict/batch`：批量预测 + Redis 同步
  - `POST /train`：模型训练（可选 lightgbm/xgboost）
  - `POST /shap/global`：全局 SHAP 解释
  - `POST /shap/local/{user_id}`：局部 SHAP 解释
  - `GET /intent/{user_id}`：从 Redis 读取意愿得分
- [x] **离线流水线脚本**：`bi/data/run_offline_pipeline.py`
  - 5 步全流程：加载数据 → 训练模型 → SHAP 全局解释 → 批量预测 → Redis 同步
- [x] **Python 单元测试**：`bi/tests/test_pipeline.py`
  - FeatureEngineer：transform、派生特征、特征列名
  - IntentPredictor：训练+预测、单用户预测
  - ShapExplainer：全局/局部解释

## 4. 给后来者的指导

- 模型训练需要先有 `user_feature_matrix` 数据，运行顺序：`generate_mock_data.py` → `run_offline_pipeline.py`
- SHAP TreeExplainer 对 LightGBM 原生支持最好，XGBoost 次之
- AUC 目标 ≥ 0.85（NFR3），若达不到可调整 mock 数据的偏置程度或增加训练数据量
- Redis Key 契约：`intent:score:{userId}` → 浮点数字符串，`intent:tier:{userId}` → HIGH/MEDIUM/LOW
