## 2. 如何启动项目（本地开发）

> 说明：本项目使用 Docker 启动 PostgreSQL/Redis；后端为 Spring Boot。  
> 常见坑：Windows 本机如果也装了 PostgreSQL，可能会占用 `5432` 端口，导致后端实际连到“本机数据库”而不是“容器数据库”。为避免冲突，建议将容器 PostgreSQL 映射到 **15432**（而非 5432）。

### 2.1 启动数据库与缓存（Docker）

在项目根目录执行：

```powershell
cd d:\Repositories\FlashSale-Pro
docker-compose up -d
```

检查容器状态（可选）：

```powershell
docker ps
```

#### 端口配置建议

- 推荐将 `docker-compose.yml` 的 PostgreSQL 端口映射设为 **`15432:5432`**（避免与本机 PostgreSQL 冲突）
- 对应地，`application.properties` 中 JDBC 连接端口也应改为 **15432**

### 2.2 启动后端（Spring Boot）

方式一：在 IDE 中直接运行主类 `FlashsaleProApplication`。

方式二：命令行运行（需要本机 Maven 环境）：

```powershell
cd d:\Repositories\FlashSale-Pro
mvn spring-boot:run
```

启动成功后，后端默认监听：`http://localhost:8080`

## 3. 如何进行测试（注册/登录）

### 3.1 用 HTTP 文件测试接口

项目提供了测试文件：`test/test.http`，包含：

- `POST /user/register`：注册新用户
- `POST /user/login`：登录已注册用户

按顺序执行：

1. 先执行注册请求，期望返回 **200**（或当用户名已存在时返回 **400**）
2. 再执行登录请求，期望返回 **200** 且 `data` 中包含 token（当用户名或密码错误时返回 **401**）

### 3.2 用 psql 验证数据是否写入（可选）

如果你 Windows 没有安装 `psql`，推荐直接在容器里执行：

```powershell
docker exec -it flashsale_postgres psql -U host -d flashsale_db
```

进入交互界面后：

```sql
\dt
SELECT * FROM "user";
```

退出：

```sql
\q
```

## 2. 项目核心框架说明
当前后端代码主要集中在 `src/main/java/com/flashsale/flashsale_pro/` 目录下，各个包的作用如下：

| 目录名称 | 角色比喻 | 核心职责说明 |
| :--- | :--- | :--- |
| **common** | **规则制定者** | 存放通用的工具类或公共结构。例如 `Result.java` 定义了后端回传数据的标准格式。 |
| **controller** | **服务员** | 控制层，负责接收前端发来的 HTTP 请求（如点击登录按钮），并将请求转交给 Service 层处理。 |
| **service** | **厨师** | 业务逻辑层，包含具体的业务规则，“大脑”所在。比如判断该用户是否已存在。 |
| **mapper** | **仓库管理员** | 数据访问层，只负责根据指令安全地同数据库交互（执行 SQL 语句）。 |
| **entity** | **食材/包装盒** | 实体类层，纯粹的“载体”对象。将数据库里干瘪的一行数据映射成带有属性的 Java 对象。 |
